# Классификация категории объявления (Avito)

## Цель и метрики
Задача: предсказать категорию объявления по тексту описания, используя классические методы машинного обучения.

Основные метрики:
- $F1_{macro}$ как основная метрика (важно не терять редкие классы).
- Accuracy как дополнительная метрика для общего контроля качества.
- Top-k accuracy и матрица ошибок для дополнительного анализа.

## Данные
Источники:
- train.csv и test.csv (датасет Avito).
- category.csvс соответствием `category_id -> name`.

Схема данных:
- Обучающая выборка: `item_id`, `title`, `description`, `price`, `category_id`.
- Тестовая выборка: `item_id`, `title`, `description`, `price`.

В модели используются только тексты `description`; `title` и `price` игнорируются.

Ограничения и качество:
- Пропуски в `description` и `price`.
- Разная длина текстов, шумные объявления.


## EDA
Проведенные исследования:
- Распределение классов и дисбаланс категорий.
- Длины `title` и `description`, частоты пустых текстов.

Выводы:
- Текст описания является основным источником сигнала.
- Название может содержать утечки, поэтому исключено из признаков.
- Цена не используется в модели.

## Валидация
- Стратифицированное разбиение train/valid.
- Дополнительно к фиксированному сплиту возможно k-fold CV для устойчивой оценки.
- Метрики считаются на валидации, итоговая модель выбирается по $F1_{macro}$

## Подход
Фичи:
- Текст: TF-IDF по `description`.

Базовая модель:
- Логистическая регрессия (мультиклассовая), как быстрый и интерпретируемый baseline.

## Обучение
Основные шаги:
- Очистка `description`.
- TF-IDF с ограничением частот (min_df) и n-gram.
- Логистическая регрессия с регуляризацией.

Подбор гиперпараметров:
- `C`, `max_features`, `ngram_range`.
- При необходимости сравнение с Linear SVM или SGDClassifier (только классические модели).

Анализ ошибок:
- Просмотр пар категорий, которые путаются чаще всего.
- Проверка ошибок на коротких и пустых текстах.

## Валидация (analysis)
- Отчет по $F1_{macro}$ , Accuracy, top-k accuracy.
- Матрица ошибок и анализ наиболее проблемных классов.
- Проверка устойчивости качества к изменениям TF-IDF и регуляризации.

## Выбор модели и риски
Финальная модель:
- Логистическая регрессия + TF-IDF по `description`.

Основные риски:
- Дисбаланс классов может снижать качество на редких категориях.
- Некачественные или пустые тексты ухудшают точность.
- Дрейф данных при изменении структуры категорий.

## Деплой
Формат применения:
- Пакетное предсказание категорий для новых объявлений.

Переобучение:
- Регулярно при накоплении новых данных или изменении каталога категорий.
